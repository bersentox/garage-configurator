# site_body

Автономный блок `site_body` для встраивания как отдельный embed-модуль.

## Назначение

Блок реализует 6-фазный UX-сценарий принятия решения:

1. История провала
2. Закономерность проблемы
3. Инвестиция в компанию
4. Инвестиция в продукт
5. Навигатор системы
6. Навигатор действия

Сценарная ось: **Страх → Понимание → Доверие → Осмысленность → Контроль → Действие**.

## Архитектурные правила

- Полная изоляция от `configurator` обязательна.
- Разработка ведётся **ТОЛЬКО** внутри `blocks/site-body/`.
- CSS использует namespace `.sb`; глобальные стили запрещены.
- JS изолирован и не взаимодействует с другими частями репозитория.
- Принцип progressive disclosure: сначала показывается структура, детали раскрываются по запросу.

## UX-принципы

- Одна фаза = одна мысль.
- Визуальная иерархия обязательна.
- Минимализм: ограниченное количество элементов.
- Интерфейс снижает неопределённость, а не «убеждает».

## Запреты

Запрещено использовать:

- лендинговые паттерны;
- визуальный шум;
- избыточные UI-элементы;
- карточки ради карточек;
- иконки без функции.

## Доступность и производительность

- Семантические `section` и корректная иерархия заголовков.
- Подготовка под клавиатурную навигацию и `:focus-visible`.
- Минимальный target size ~24px для интерактивных элементов.
- Контраст с ориентацией на WCAG AA.
- Без библиотек и с минимальным JS.
- Структура должна быть готова к Core Web Vitals (LCP/CLS/INP).
