# site_body

Технический документ для автономного блока `site_body` в `blocks/site-body/`.

## Архитектура

- Корневой контейнер: `<section id="site-body" class="sb">`.
- Внутри ровно 6 последовательных фаз:
  - `<section data-phase="1">`
  - `<section data-phase="2">`
  - `<section data-phase="3">`
  - `<section data-phase="4">`
  - `<section data-phase="5">`
  - `<section data-phase="6">`
- Изоляция стилей: только namespace `.sb` и его элементы.
- Изоляция логики: `site-body.js` работает только с DOM внутри `#site-body`.

## Фазы и назначение

1. **Контекст проблемы**
   - Состав: media + текст.
   - Цель: показать исходную боль и цену ошибки.
2. **Аналитика повторяемости**
   - Состав: аналитический блок с метрикой.
   - Цель: подтвердить, что проблема системная.
3. **Опора на компанию**
   - Состав: grid из критериев (4 пункта).
   - Цель: зафиксировать основания доверия к процессу.
4. **Опора на продукт**
   - Состав: список + схема.
   - Цель: перевести выбор в структуру критериев.
5. **Navigator системы**
   - Состав: timeline + disclosure.
   - Цель: progressive disclosure (сначала маршрут, затем детали).
6. **Переход к действию**
   - Состав: шаги + CTA.
   - Цель: завершить сценарий конкретным следующим действием.

## Navigator (фаза 5)

- Timeline показывает базовый маршрут без перегрузки.
- Disclosure раскрывает детали по кнопке.
- Контракт доступности:
  - `<button>` как триггер;
  - `aria-expanded` отражает текущее состояние;
  - скрытый блок управляется через `hidden`.

## Правила разработки

- Разработка ведётся только в:
  - `site-body.html`
  - `site-body.css`
  - `site-body.js`
  - `README.md`
- Нельзя менять структуру директории `blocks/site-body/`.
- Нельзя подключать внешние библиотеки.
- Нельзя добавлять глобальные стили или зависимости от других блоков.
- Дизайн: минимализм, одна мысль на экран, без декоративного шума.

## Доступность и производительность

- Семантические секции и связка заголовков через `aria-labelledby`.
- Поддержка keyboard navigation через нативные элементы.
- Явный `:focus-visible` для интерактивов.
- Минимальный интерактивный target size через токены.
- Минимальный JS без внешних библиотек, без layout shift на раскрытии.
